기본구조
클라이언트(프런트) => POST, GET 요청 + json 바디 => 웹서버 (nginx) => django(백엔드)가 처리 => 리턴 or DB 저장

필요한 백엔드 API
1. 유저 등록 == /register로 받은 저거는 
2. 유저 로그인 == 받은 이메일이 DB에 있는지 그리고 비밀번호가 DB의 비밀번호와 동일여부, 쿠키 발행
3. 비밀번호 암호화
4. 토큰 생성
5. 토큰 로그인 == 쿠키에서 토큰 가져오기, DB와 대조, 로그아웃 시 토큰 삭제


6. 아이템 등록
7. 아이템 표시
8. 유저 정보 표시
9. 평점 등록
10. 유저 평점 목록 표시
11. 유저 판매 목록 표시
12. 유저 채팅 표시
13. 채팅
14. 검색바 검색
15. 조건 검색
16. 지도 검색

DB에 최종 로그인 시각 필요할 듯 ==> 일정 시간 이후 토큰 삭제 용도
